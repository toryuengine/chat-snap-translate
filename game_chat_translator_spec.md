# ゲームチャット翻訳ツール 仕様書

バージョン: 1.1  
作成日: 2025年

---

## 1. 概要

ゲーム（OverWatchなど）プレイ中に、外国語のチャットテキストをショートカットキー1操作で即座にキャプチャし、生成AIによるOCRと翻訳を経て、軽量なオーバーレイGUIに日本語訳を表示するデスクトップツール。

アプリは常駐プロセスとして起動し続け、グローバルホットキーをバインドした状態で待機する。

---

## 2. プロセスのライフサイクル

```
アプリ起動
    ↓
初期化（設定読み込み・ホットキーバインド・GUIオブジェクト生成）
    ↓
【待機状態】←─────────────────────────────┐
    │ ホットキー押下                         │
    ↓                                        │
スクリーンキャプチャ（クロップ）              │
    ↓                                        │
Anthropic API呼び出し（OCR + 翻訳）          │
    ↓                                        │
翻訳結果をGUIに表示                          │
    ↓                                        │
自動フェードアウト後、待機状態に戻る ─────────┘

    ※ タスクトレイアイコンから終了
```

---

## 3. 待機状態の仕様

- アプリ起動後、タスクトレイにアイコンを常駐させる
- ホットキーはアプリ起動時に即バインドし、ゲームにフォーカスがある状態でも受け付ける（グローバルフック）
- 待機中のCPU使用率: 実質0%（イベント駆動のため、ポーリングしない）
- 待機中のメモリ使用量: 50MB 以下
- GUIウィンドウは非表示のまま待機（翻訳結果が出た時点で表示）

---

## 4. ホットキー処理

- デフォルトキー: `Ctrl + Shift + T`（設定で変更可能）
- バインド方式: OSレベルのグローバルキーフック（`keyboard` ライブラリ）
- キー押下を検知した瞬間にキャプチャを開始する
- ホットキー検知からAPIリクエスト送信まで: 200ms 以内
- 翻訳処理中に再度ホットキーが押された場合: 処理中リクエストをキャンセルし、新しいキャプチャで再実行する

---

## 5. スクリーンキャプチャ

- キャプチャ対象エリア（チャットエリアの座標・サイズ）は設定ファイルに保存済みの固定座標を使用
- ライブラリ: `mss`（軽量・高速、DirectX画面にも対応）
- キャプチャ解像度: 元スクリーン解像度で取得後、API送信前に最大幅800pxにリサイズ

---

## 6. OCR・翻訳処理（生成AI API）

- 使用API: Anthropic Claude API（`claude-sonnet-4-20250514`）
- 入力: キャプチャ画像（base64エンコード）
- 処理内容: OCRと翻訳を1リクエストで完結させる（往復コストを削減）
- タイムアウト: 8秒

**プロンプト方針:**
- 画像内のテキストを全て読み取る
- 検出した言語を識別し、日本語訳を出力する
- 日本語テキストはそのまま返す（翻訳不要）
- レスポンスはJSON形式で返す

**APIレスポンス形式:**
```json
{
  "lines": [
    {
      "original": "GG no re",
      "language": "en",
      "translation": "お疲れ様でした"
    }
  ]
}
```

---

## 7. 表示GUIの仕様

**ウィンドウ属性:**
- 常に最前面表示（AlwaysOnTop）
- フレームレス・半透明背景（透明度85%、設定可能）
- 翻訳結果が出るまで非表示のまま待機

**レイアウト:**
- サイズ: 300〜500px幅、内容に応じて縦方向に自動伸縮
- 配置: 画面右下固定（デフォルト）、ドラッグで移動可能

**表示内容:**
- 翻訳中はローディングインジケーターをGUI上に表示
- 翻訳行ごとに「元テキスト（小・グレー）」と「日本語訳（白・大）」をペア表示
- 最大5行まで表示。超過分はスクロール可能
- 表示後10秒で自動フェードアウト（設定可能）
- フェードアウト後はウィンドウを非表示に戻す（プロセスは維持）

**表示例:**
```
┌─────────────────────────────────┐
│  GG no re                       │
│  → お疲れ様でした               │
│─────────────────────────────────│
│  Nice shot!                     │
│  → ナイスショット！             │
└─────────────────────────────────┘
```

---

## 8. タスクトレイ

- 常駐中はタスクトレイにアイコンを表示
- 右クリックメニュー:
  - 設定を開く
  - キャプチャエリアを再設定
  - 終了

---

## 9. 設定項目

| 項目 | 内容 | デフォルト値 |
|------|------|-------------|
| ホットキー | 起動キーの組み合わせ | Ctrl+Shift+T |
| キャプチャエリア | 座標(X, Y, W, H) | セットアップ時に設定 |
| 表示時間 | 自動フェードアウトまでの秒数 | 10秒 |
| ウィンドウ透明度 | GUIの背景透明度 | 85% |
| ウィンドウ位置 | 画面上の表示位置 | 右下 |
| APIキー | Anthropic APIキー | （必須入力） |

設定はローカルファイル（`config.json`）に保存。

---

## 10. 初回セットアップ

1. アプリを起動する
2. Anthropic APIキーを入力する
3. キャプチャエリア設定モードに入り、ゲーム画面のチャット部分をドラッグ選択する
4. 設定を保存 → 待機状態に移行

---

## 11. 技術スタック（推奨）

| 用途 | 推奨技術 |
|------|---------|
| アプリ本体 | Python 3.10+ |
| グローバルホットキー | `keyboard` ライブラリ |
| スクリーンキャプチャ | `mss` |
| GUI | `tkinter`（標準ライブラリ） |
| タスクトレイ常駐 | `pystray` |
| API通信 | `anthropic` SDK または `httpx` |
| 設定ファイル | JSON |

---

## 12. 非機能要件

- 待機中CPU使用率: 実質0%（イベント駆動）
- 待機中メモリ: 50MB 以下
- ホットキー押下からAPI送信まで: 200ms 以内
- ホットキー押下から翻訳表示まで: 通常8秒以内（API通信込み）
- キャプチャ・翻訳処理は別スレッドで実行し、ゲームパフォーマンスに影響させない

---

## 13. 対象外・制限事項

- リアルタイム自動監視は対象外（手動トリガーのみ）
- 装飾の多いフォントはOCR精度が低下することがある
- Windows 10/11 を主対象（macOS / Linux は対象外）
